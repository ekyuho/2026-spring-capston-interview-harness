기말에 대략 어떤 것을 만들어 보여줄지의 개요, 그 의미:

- [구현 목표 범위] AI가 소비를 예측하고, 오늘의 챌린지를 자동 생성하는 핵심 흐름을 실제로 동작하는 앱으로 보여주는 것을 목표로 합니다.
- [Core 기능] - 회원가입 / 로그인 - 온보딩: 카테고리 선택 + 월 예산 설정 - 더미 소비 데이터 기반 Prophet 예측 → "이 페이스면 이번 달 카페에 N원 쓸 것 같아요" - 데일리 챌린지 자동 생성 및 화면 표시 (난이도 + 챌린지 텍스트 + XP) - 소비 수기 입력 및 내역 조회 - (여유가 있을 시) 챌린지 달성 시 캐릭터 경험치 획득 및 레벨업 [의미] 기술적 의미: 이번 실험 코드에서 증명한 `예측 → 행동 판단 → 챌린지 생성` 파이프라인이 실제 앱 안에서 동작한다는 것을 보여줍니다.
- 지금까지는 Colab 안에서 더미 데이터로 돌아가던 로직이, 기말에는 Kotlin으로 구현된 화면과 FastAPI 서버, MySQL DB가 연결된 실제 서비스 형태로 작동하게 됩니다.
- 한계와 확장 가능성: 기말 시점에서는 더미 데이터와 수기 입력을 사용합니다.
- 하지만 계좌 연동이 가능해지면 신규 유저도 가입 즉시 과거 소비 이력을 기반으로 정교한 예측을 받을 수 있고, 유저가 쌓이면 개인 데이터만이 아닌 유사 소비 패턴 유저 그룹의 집계 데이터를 활용한 협업 필터링 방식으로 예측 정확도를 높일 수 있습니다.
- 기말 MVP는 그 확장의 출발점이 되는 핵심 파이프라인을 검증하는 단계입니다.

서비스를 지향하는 산학트랙의 경우, API Call을 통해 어떤 AI 서비스를, 어떤 목적으로, 어떤 파라미터를 주고, 그 결과를 어떻게 활용하는지 간단히 정리(어떻든 AI를 적극적으로 적용해보세요):
- OpenAI API: `gpt-4o-mini` 모델 사용 예정 [목적] Prophet이 예측값과 난이도(Easy/Medium/Hard)를 결정하면, 그 결과를 바탕으로 사용자에게 보여줄 챌린지 텍스트를 자연어로 생성하는 데 사용합니다. 규칙 기반으로도 문장을 만들 수 있지만, 유저의 소비 맥락(어제 얼마 썼는지, 어떤 카테고리인지, streak 상황 등)을 반영한 개인화된 문장을 자동으로 만드는 것이 목적입니다. [파라미터] System prompt: "당신은 소비 습관 코칭 앱 Moni의 챌린지 생성기입니다. 사용자의 소비 데이터를 바탕으로 오늘 실천할 수 있는 구체적이고 친근한 챌린지 문장을 한 문장으로 생성하세요." User prompt (동적으로 채워지는 값): [활용] API 호출 실패 시에는 규칙 기반 fallback 문장이 대신 표시되어 앱이 중단되지 않도록 설계합니다.
- 챌린지 문장 개인화 생성
- 카테고리: 카페
- 어제 소비: 8,500원
- 최근 7일 평균: 5,200원
- 예산 대비 소비 비율: 92%
- 난이도: Medium
- 무지출 streak: 0일
- API 응답으로 받은 챌린지 문장을 앱 홈 화면의 챌린지 카드에 표시합니다. Prophet이 계산한 난이도, XP, 생성 이유는 별도로 함께 표시되고, OpenAI는 그 난이도에 맞는 자연스러운 문장 생성을 담당합니다.

- **이번에 팀이 구성된 계기**: 에타

### 박은수

- **본인의 기술적 관심영역,  실제로 만들어서 작동시켜본 산출물, 남보다 잘한다고 생각되는 영역 등 소개**: 프론트엔드 및 디자인에 관심이 있으며, html을 기반으로 하는 간단한 문서 정리 프로그램을 제작, 혹은 이미 올려진 오픈 소스를 수정해본 경험이 있음. 또한 cloud flare와 github를 이용하여 인터넷 웹서버를 통해 접속할 수 있는 개인용의 소규모 사이트를 배포해본 경험이 있음(AI를 이용한 시뮬레이션 프로그램). 포토샵, 애프터 이펙트 등의 비주얼 툴을 능숙하게 다룰 수 있음.
- **이번에 팀이 구성된 계기**: 비슷한 개발 수준을 가진 팀원끼리 학습을 통해 상호 발전을 도모하기로 결정

### 신희조

- **본인의 기술적 관심영역,  실제로 만들어서 작동시켜본 산출물, 남보다 잘한다고 생각되는 영역 등 소개**: 유니티를 활용해 게임 개발 프로젝트를 두 번 진행해본 경험이 있고, 무료로 플레이 가능한 웹사이트에 업로드 해보았음. 학교 수업에서 자바 spring, 웹소켓을 활용한 멀티 카드 게임을 개발해봤음.
- **이번에 팀이 구성된 계기**: 에브리타임 어플에서 팀원을 구하여 구성됨.

## 5. 개인별 5 Questions

### 신희조

1. 현재 소비/지출 관리를 어떻게 하고 있는가
2. 예산을 정했을 때 그 범위 내에서 소비/지출이 잘 지켜졌는가
3. 소비 관리를 할 때 가장 번거롭거나 불편한 점이 있다면 무엇인가
4. 소비 관련 앱(가계부 등)을 사용하다가 그만둔 적이 있는지, 있다면 이유는 무엇인지
5. 부적절한 소비 습관을 바꾸는데에 효과적인 방법이 무엇이라고 생각하는가

### 윤수연

1. 지금 쓰는 가계부 앱이 있나요? 있다면 아쉽거나 불편한 점이 있나요?
2. 월말에 돈이 부족했던 경험이 있나요? 왜 그랬던 것 같나요?
3. 소비 관리에서 제일 힘든 게 뭐예요?
4. AI가 "이번 달 카페 소비가 너무 많아요."라고 말해주면 행동이 바뀔 것 같나요? 아니라면 어때야 바뀔 것 같나요?
5. "지금 내 상황에서 뭘 줄여야 돼?"라고 물어볼 수 있는 기능이 있으면 쓸 것 같나요?

### 박은수

가계부나 앱을 사용해본 적이 있나요? 사용한다면 어떤 점이 불편했나요? 아예 사용하지 않는다면 이유는 무엇인가요?
예산을 세워도 지키기 어려운 이유는 무엇인가요?
소비를 돌아볼 때 어떤 점이 가장 부족하다고 느끼나요? (예: 기록, 분석, 피드백 등)
누군가가 소비에 대해 피드백을 준다면 도움이 될 것 같나요?
소비 기록이 얼마나 간단해야 계속 사용할 수 있을까요?
