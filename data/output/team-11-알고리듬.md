기말에 대략 어떤 것을 만들어 보여줄지의 개요, 그 의미:
- 백엔드 스케줄링 엔진과 프론트엔드 대시보드를 완전히 결합하여, 교사가 자연어로 요구사항을 입력하면 AI가 이를 즉시 분석해 수정된 시간표를 그려주는 풀스택 MVP를 완성할 계획 단순히 시간표를 한 번 짜고 끝내는 것이 아니라, 학기 중 발생하는 보결이나 수업 교환 요청을 시스템이 실시간으로 검증하고 업데이트하는 전체 운영 프로세스를 시연하는 것이 목표 이는 수작업 위주의 경직된 학사 행정을 데이터 기반의 자동화 시스템으로 전환하여 교사들의 행정 부담과 감정 소모를 획기적으로 줄이는 데 가장 큰 의미가 있음. 교육 현장에서 발생할 수 있는 돌발 변수들에 즉각적으로 대응할 수 있는 지능형 학사 운영의 표준 모델을 제시하고자 함

서비스를 지향하는 산학트랙의 경우, API Call을 통해 어떤 AI 서비스를, 어떤 목적으로, 어떤 파라미터를 주고, 그 결과를 어떻게 활용하는지 간단히 정리(어떻든 AI를 적극적으로 적용해보세요):
- 목적: 교사의 비정형 요구사항을 스케줄링 엔진용 JSON 제약조건 데이터로 정규화 주요
- 파라미터: System Prompt: 자연어 문장을 제약조건 리스트(JSON)로 변환하는 규칙 정의, Temperature: 0.0 (데이터 무결성을 위해 일관된 결과 도출) 결과
- 활용: 추출된 제약조건(요일, 교시, 가중치 등)을 유전 알고리즘의 평가 지표로 즉시 산입하여 최적의 시간표 생성

AI 투명성 리포트 (텍스트로 간단히 정리):
- OpenAI GPT-4o API를 활용해 교사들의 복잡한 요구사항을 시스템용 데이터로 변환하는 데 AI를 사용함 사용자가 입력한 비정형 텍스트에서 제약조건 항목과 가중치만 정확히 뽑아내는 전처리 파이프라인 역할에 집중함 데이터 처리의 투명성을 위해 개인정보는 제외하고 수업 관련 정보만 입력받으며, 템퍼러처를 0으로 고정해 AI의 자의적 해석이나 환각 현상을 원천 차단함 최종적으로 AI가 추출한 결과값을 사용자가 직접 확인하고 수정할 수 있는 단계를 두어 인간의 개입과 통제권을 보장하며, 특정인에게 유리한 결과가 나오지 않도록 누적 시수 같은 객관적 지표를 알고리즘에 함께 녹여 공정성을 유지함
