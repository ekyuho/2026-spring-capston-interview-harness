기말에 대략 어떤 것을 만들어 보여줄지의 개요, 그 의미:
- 개요: 사진 촬영/업로드 → YOLO: 음식 사진인지 1차 판별 → Gemini: 음식 인식 + 칼로리 + 나트륨 등 평가 → 결과를 DB 저장(식단 히스토리) → 대시보드: 주간 나트륨 섭취 그래프, 위험 음식 빈도, 개선 추이 등 → 개인화 알림
- 의미: 만성질환 식단 관리는 단순히 칼로리를 줄이는 것을 넘어, 질환별로 제한해야 할 성분(나트륨, 당, 단백질 등)이 제각각이라 환자 본인이 직접 관리하기가 매우 까다롭습니다. 이 시스템은 "사진 한 장"이라는 최소한의 행동으로 전문 영양사 수준의 피드백을 제공함으로써, 만성질환 환자의 식단 자기관리 접근성을 획기적으로 낮추는 것을 목표로 합니다.

서비스를 지향하는 산학트랙의 경우, API Call을 통해 어떤 AI 서비스를, 어떤 목적으로, 어떤 파라미터를 주고, 그 결과를 어떻게 활용하는지 간단히 정리(어떻든 AI를 적극적으로 적용해보세요):
- 어떤 AI 서비스를: Google Gemini Flash API (멀티모달 LLM)
- 어떤 목적으로: 고혈압 환자가 촬영한 음식 사진을 분석하여 식단 구성, 칼로리 추정, 나트륨 위험도 평가 및 개선 제안 제공
- 어떤 파라미터를
- 모델: 'gemini-2.5-flash-preview'
- 입력: 음식 이미지(PIL Image) + 텍스트 프롬프트
- 프롬프트 핵심 지시: 전문 영양사 페르소나 부여, DASH 식단 기준 평가, 한국어 출력, 3섹션 구조화 형식 지정
- 결과를 어떻게 활용
- 식단 구성 목록 → 사용자에게 음식 인식 결과 표시
- 칼로리 추정값 → 일일 섭취 칼로리 관리에 활용
- 식단 평가 → 나트륨 경고 및 구체적 식습관 개선 조언 제공

AI 투명성 리포트 (텍스트로 간단히 정리):
- 사용 AI 모델: 본 프로젝트는 두 가지 AI 모델을 활용합니다. 1차 판별에는 Ultralytics의 오픈소스 객체 탐지 모델인 YOLOv8을 사용하여 입력 이미지가 음식인지 여부를 판단합니다. 2차 분석에는 Google DeepMind의 Gemini Flash 멀티모달 LLM을 API 형태로 사용하여 음식 인식, 칼로리 추정, 식단 평가를 수행합니다.
- 팀의 기여 범위: YOLO와 Gemini 모델 자체는 팀이 개발한 것이 아닙니다. 팀의 실질적 기여는 두 모델을 연결하는 파이프라인 설계, 고혈압 도메인에 특화된 프롬프트 엔지니어링, 그리고 DASH 식단 기준을 반영한 출력 구조 설계에 있습니다.
- 의료적 면책: 본 시스템의 출력은 의료적 진단이나 처방을 대체하지 않습니다. 고혈압 치료 및 식단 계획은 반드시 담당 의사 및 영양사와 상담하시기 바랍니다.
- 데이터 처리: 업로드된 이미지는 Google Gemini API 서버로 전송되며, Google의 이용약관 및 개인정보처리방침이 적용됩니다. 현재 데모 단계에서는 이미지를 별도로 저장하지 않습니다.

## 5. 기반 소프트웨어 리스트

- **env**: 1. Client 디바이스는 Mobile (iOS/Android 통합 앱)
2. FE는 Flutter (Dart)
3. BE는 FastAPI (Python)
4. DB는 MySQL (사용자 정보 및 식단/운동 로그 저장)
5. FE 또는 BE에 사용하는 특별한 라이브러리
      - AI/DL: PyTorch(딥러닝 엔진), Ultralytics (YOLOv8/v10 모델 라이브러리)
      - BE: SQLAlchemy (ORM), Pydantic (데이터 검증), Uvicorn (ASGI 서버)
      - DATA: Pandas/NumPy (식단 영양소 데이터 처리)
6. API Call로 사용할 서비스는 OpenAI API (GPT-4 기반 건강 상담 챗봇), 공공데이터포털 API (식품 영양 성분 데이터베이스)
