## 5. 개인별 5 Questions

### 고윤진

- **질문당 한줄씩 5개의 질문**: 1. 현재 운영 중인 서비스에서 LLM API 비용을 모니터링하거나 예측하기 위해 별도의 툴(대시보드, 엑셀 등)을 사용하고 계신가요? 사용하면서 불편하셨던 점은 무엇인가요?
2. GPT-4, Claude, Gemini 등 다양한 모델 사이에서 의사결정을 내릴 때, '성능'과 '비용' 중 어느 요소에 더 큰 비중을 두시나요?
3. 설정한 예산을 초과할 경우 즉시 알림(Slack, 이메일 등)을 받거나, 모델별 사용 비중을 한눈에 시각화해 주는 리포트 기능 중 어떤 것이 더 우선적으로 필요하신가요?
4. 만약 이 서비스가 프롬프트 최적화를 통해 전체 API 비용의 30~50%를 절감해 준다면, 해당 서비스 이용을 위해 매달 일정 금액 구독료를 을 지불할 용의가 있으신가요?
5. 신규 서비스 런칭 전, 예상 사용자 수와 호출 빈도를 입력했을 때 월간 API 지출액을 미리 시뮬레이션해 보는 기능이 비즈니스 의사결정에 얼마나 도움이 될 것이라고 생각하시나요?

### 김나현

- **질문당 한줄씩 5개의 질문**: 평소에 LLM 관련 지출이 부담스럽게 느껴진 적이 있습니까?
LLM 관련하여 한 달 평균 어느 정도 지출합니까?
토큰을 약 50% 줄여주는 유료 서비스가 있다면, 얼마까지 낼 의향이 있습니까?
가장 많이 사용하는 LLM은 어떤 모델이며, 그 이유는 무엇입니까?
몇 가지 정도의 LLM을 사용하십니까? 그 모델은 무엇입니까?

### 이유진

- **질문당 한줄씩 5개의 질문**: 현재 LLM API를 사용하실 때, 월 비용이 얼마나 나오는지 파악하고 계신가요? 비용을 추적하는 별도의 방법이 있으신가요?
프롬프트를 작성할 때 토큰 수나 비용을 의식하면서 작성하시나요, 아니면 일단 결과 품질 위주로 작성하시나요?
LLM 모델을 선택할 때 가장 중요하게 고려하는 기준은 무엇인가요? (예: 가격, 응답 품질, 속도, 한국어 성능 등)
같은 작업에 대해 여러 LLM 모델의 비용이나 성능을 비교해본 경험이 있으신가요? 있다면 어떤 방식으로 비교하셨나요?
만약 프롬프트를 자동으로 줄여서 비용을 절감해주는 도구가 있다면, 결과 품질이 몇 % 정도까지 유지되어야 사용하실 의향이 있으신가요?

## 6. 기반 소프트웨어 리스트

env:
  - 1. Client 디바이스: PC(Windows, Mac)
  - 2. FE:React.js ,TailwindCSS
  - 3. BE:FastAPI (Python)
  - 4. DB: Firebase Firestore
  - 5. AI/LLM: tiktoken
  - 6. 외부 API: OpenAI / Anthropic / Google AI SDK
  - 7.크롤러: BeautifulSoup
  - 8. 배포: Vercel (프론트) + Railway or Render (백엔드)
