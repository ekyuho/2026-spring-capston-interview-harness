# AGENTS.md

## 목적
이 저장소는 이화여대 컴퓨터공학 캡스톤디자인 심층인터뷰 준비를 위해 GitHub와 Google Spreadsheet에 흩어진 자료를 팀별로 통합하여 팀별 인터뷰 준비 문서(Dossier)를 자동 생성한다.

## 자동화 목표
1. GitHub `project briefs.md`에서 팀별 프로젝트 개요를 읽는다.
2. Google Sheets에서 다음 자료를 읽는다.
   - 심층인터뷰 보충자료
   - 학기초 개인설문
   - 개인별 5 Questions
   - 기반 소프트웨어 리스트
3. 팀번호와 학생 이름을 기준으로 자료를 병합한다.
4. 팀별 Markdown 인터뷰 준비서를 생성한다.
5. 병합 실패 또는 이름 불일치 항목은 `data/output/unresolved.md`에 남긴다.

## 핵심 단위
- Team: 팀번호, 팀명, 트랙, 프로젝트명, GitHub repo
- Member: 학생 이름, 소속 팀, 개인 설문, 개인 질문
- Dossier: 팀별 심층인터뷰 준비서

## 병합 규칙
1. 팀 단위 자료는 팀번호를 우선 키로 사용한다.
2. 개인 단위 자료는 학생 이름을 기준으로 팀에 붙인다.
3. 팀번호와 이름이 모두 불명확하면 자동 병합하지 않는다.
4. 자동 병합 실패 항목은 반드시 `unresolved.md`에 기록한다.
5. 개인정보성 응답은 외부 공개 문서에 그대로 노출하지 말고 교수자 검토용으로만 사용한다.

## 첫 구현 범위
처음에는 병합까지 하지 말고, 데이터 로딩과 컬럼 진단까지만 구현한다.

생성해야 할 1차 결과물:
- `data/raw/*.csv`: 각 원본 시트 저장
- `data/output/debug_columns.md`: 각 자료의 컬럼명과 샘플 3행

## 이후 구현 범위
1. 팀/학생 정규화 테이블 생성
2. 팀별 자료 병합
3. 팀별 dossier Markdown 생성
4. 인터뷰 질문 초안 자동 생성

## 코딩 규칙
- Python 3.11 이상 기준
- `pandas`, `requests`, `PyYAML`, `Jinja2` 사용
- 네트워크 오류와 빈 시트는 예외 처리
- 파일 인코딩은 UTF-8
- Google Sheet는 우선 CSV export URL 방식으로 읽는다.
